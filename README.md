# آپشن یار — اپلیکیشن بازار آپشن و بورس کالای ایران

اپلیکیشن اندروید (Kotlin + Jetpack Compose) که **بدون VPN در ایران کار می‌کند** (هیچ وابستگی به سرویس‌های گوگل/Firebase ندارد):

- ✅ **ورود با شماره موبایل (کد پیامکی)** — بدون گوگل، بدون VPN. هر شماره یک حساب جدا.
- ✅ بازار آپشن بورس ایران — رایگان — قیمت زنده + رتبه‌بندی به‌صرفه‌ترین قراردادها با استراتژی کاورد کال
- ✅ **فرمول‌یاب سود هدف** — کاربر سود موردنظر (مثلاً ۲۰٪) و بازه (۱/۲/۳ ماه) را می‌دهد و اپ قراردادهای مناسب را با استدلال معرفی می‌کند
- ✅ **زنجیره قرارداد هر سهم** — همه قراردادهای هر سهم پایه، گروه‌بندی‌شده و مرتب بر اساس نوع و قیمت اعمال
- ✅ **پروفایل کاربر** — نام و شماره، وضعیت اشتراک، و دسترسی به اشتراک/تاریخچه/سوالات/خروج
- ✅ **سوالات و انتقادات** — ارسال پیام به پشتیبانی و نمایش پاسخ ادمین
- ✅ بازار بورس کالا — فقط برای مشترکین، با قفل و دکمه خرید اشتراک
- ✅ صفحه اشتراک: شماره کارت + فعال‌سازی با «کد فعال‌سازی» (بدون درگاه/گوگل)
- ✅ نمودار سود و زیان استراتژی کاورد کال
- ✅ تاریخچه فعالیت که روی خود گوشی ذخیره می‌شود
- ✅ یک GitHub Action که خودش APK را می‌سازد

## معماری (چرا بدون VPN کار می‌کند)

| بخش | سرویس | داخل ایران |
|------|-------|-----------|
| داده‌ی بازار (آپشن/کالا) | BrsApi.ir (ایرانی) | ✅ بدون VPN |
| ورود، اشتراک، سوالات | **بک‌اند خودمان** (`server/`) روی هاست ایرانی | ✅ بدون VPN |

قبلاً این اپ برای ورود از Google Sign-In و برای داده از Firebase استفاده می‌کرد که در ایران فیلترند و VPN می‌خواستند. حالا هر دو با سرویس‌های ایرانی جایگزین شده‌اند.

## منبع داده بازار

- **بازار آپشن**: داده زنده از BrsApi (`https://BrsApi.ir/Api/Tsetmc/Option.php`) — هر ۲۰ ثانیه به‌روزرسانی.
- **بازار کالا**: قیمت لحظه‌ای طلا و سکه از BrsApi (`https://BrsApi.ir/Api/Market/Gold_Currency.php`).

هر دو در `BrsApiMarketRepository.kt` پیاده شده‌اند. کلید API باید در سکرت `BRSAPI_KEY` باشد (GitHub Settings → Secrets → Actions). اگر کلید نباشد یا سرویس پاسخ ندهد، صفحه به‌جای اسپینر بی‌پایان، **علت دقیق** را نشان می‌دهد (کلید نیست / خطای سرور / بازار بسته / خطای شبکه).

## بک‌اند (ورود موبایلی، اشتراک، سوالات)

کد بک‌اند در پوشه‌ی `server/` است (Node.js). راهنمای کامل اجرا و دیپلوی (لیارا یا VPS ایرانی) و اتصال پنل پیامک (کاوه‌نگار/SMS.ir) در `server/README.md` آمده است.

بعد از دیپلوی، آدرس عمومی بک‌اند را در سکرت `BACKEND_URL` بگذار (GitHub Settings → Secrets → Actions) و دوباره APK بساز. اگر این سکرت خالی باشد، اپ هنگام ورود پیام «آدرس سرور تنظیم نشده» می‌دهد.

## سکرت‌های لازم برای build

| سکرت | برای چه | نبودنش چه می‌شود |
|------|---------|-----------------|
| `BRSAPI_KEY` | داده‌ی بازار | بازارها خالی + پیام «کلید تنظیم نشده» |
| `BACKEND_URL` | ورود/اشتراک/سوالات | ورود کار نمی‌کند + پیام مربوطه |

## گرفتن فایل APK نصبی

بعد از هر push به شاخه main (یا اجرای دستی از تب Actions):

1. تب **Actions** ریپازیتوری را باز کن
2. آخرین اجرای «Build Debug APK» را باز کن و صبر کن سبز شود
3. پایین صفحه، بخش **Artifacts** → فایل `covered-call-app-debug-apk` را دانلود کن
4. فایل zip را باز کن و APK داخلش را نصب کن

## اشتراک بورس کالا (بدون درگاه پرداخت)

۱. کاربر مبلغ را به کارت واریز می‌کند و رسید را به پشتیبانی (تلگرام، در `SubscriptionScreen.kt` قابل تغییر) می‌فرستد.
۲. ادمین یک **کد فعال‌سازی** می‌سازد و به کاربر می‌دهد:
   - از خط فرمان: `cd server && npm run gen-codes 5 30` (۵ کد ۳۰روزه)
   - یا با endpoint ادمین `/admin/activation-codes` (به `server/README.md` نگاه کن)
۳. کاربر کد را در صفحه‌ی اشتراک وارد می‌کند و همان لحظه اشتراکش فعال می‌شود.

برای فعال‌سازی مستقیم یک شماره هم `/admin/subscription` هست.

## پاسخ به سوالات و انتقادات کاربران

پیام‌های کاربران در بک‌اند ذخیره می‌شوند. برای دیدن و پاسخ، از endpointهای ادمین استفاده کن:
```
GET  /admin/feedback
POST /admin/feedback/reply   {"id":"...","reply":"..."}
```
پاسخ همان لحظه در اپ کاربر زیر پیامش نمایش داده می‌شود. جزئیات در `server/README.md`.
